# OnO Explain

## Workflow

1. 추측으로 설명하지 않고 관련 파일과 호출 경로를 먼저 확인한다.
2. 설명은 `요청 진입점 -> 핵심 로직 -> DB/API/외부 연동 -> 응답/상태 반영` 순서로 정리한다.
3. 중요한 근거는 파일 경로와 라인으로 연결한다.
4. 백엔드 개발자 관점에서 API 계약, DTO, validation, 예외 처리, 트랜잭션, 보안, 성능 포인트를 짧게 짚는다.
5. 개선 제안은 현재 코드의 실제 리스크가 있을 때만, tradeoff와 함께 제시한다.

## Output

- 먼저 전체 흐름을 짧게 요약한다.
- 그다음 핵심 파일별 역할을 설명한다.
- 마지막에 주의할 운영 리스크나 학습 포인트를 정리한다.
