# /analysis — 원인 규명·분석

**읽기 전용 명령: 코드 수정·커밋 절대 금지.**

## 시작

인자($ARGUMENTS)가 없거나 불명확하면, 먼저 아래 3가지를 물어보고 답을 받은 뒤 진행:
1. 무슨 문제가 발생했나? (증상·에러·예상과 다른 동작)
2. 원하는 결과는 무엇인가?
3. 관련 파일·경로·로그가 있으면 알려달라.

## 조사 단계

1. **조사각도 분해** — 이 문제를 설명할 수 있는 가설 3~5개를 먼저 나열한다.
2. **병렬 조사** — `code-tracer` 에이전트와 필요 시 `permission-auditor`, `perf-reviewer`, `logic-reviewer`, `db-safety` 에이전트를 병렬로 투입해 각 가설을 독립적으로 검증한다.  
   - 각 에이전트는 Read, Grep, Glob, Bash(git log/diff/show) 도구만 사용.
   - 근거는 반드시 `파일:라인` 형식으로 명시. 추정 금지.
3. **교차 반증** — 각 에이전트 결과를 서로 교차 검증: 가설 A를 지지하는 증거가 가설 B를 반박하는가? 모순·공백 식별.
4. **종합** — 확신도(높음/중간/낮음)와 함께 근본 원인 결론 도출.

## 산출물

분석 보고서 (텍스트, 저장 불필요):
- **요약**: 한 문장 결론
- **근거**: 파일:라인 목록
- **근본 원인 + 확신도**
- **다음 단계 제안** (plan 또는 추가 조사 방향)
