# /feat — 기능 구현

## 작업: $ARGUMENTS

## 단계

1. **조사 & 계획 확인** — 관련 계획서(`docs/plans/`)가 있으면 읽는다. 없으면 `code-tracer` 에이전트로 영향 범위 파악 후 간략 계획을 제시하고 **승인 대기**.

2. **승인 확인** — 계획이 승인되지 않았다면 구현 시작 금지. "계획을 확인해주세요." 안내.

3. **구현** — 승인된 계획에 따라 구현:
   - 엔티티 변경 시 `./gradlew compileJava`로 컴파일 확인.
   - 가드레일 준수: FCM 경로·프로덕션 DB 파괴 작업 전 경고.
   - 사용자 격리(userId 소유권 검증) 누락 없이 구현.
   - 근거 없는 코드 추가 금지. 각 변경은 계획서 단계에 대응.

4. **컴파일 확인** — 구현 완료 후 `./gradlew compileJava` 실행. 오류 있으면 수정.

5. **커밋 제안** — 논리 단위별 커밋 메시지 초안을 제시하고 사용자 확인 후 커밋.  
   형식: `[Feat] 한 줄 요약\n- 상세`  
   (테스트는 /test로 분리)

> 🟠 구현 완료 후 /test → /check 순서로 검증 권장.
