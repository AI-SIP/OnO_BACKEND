# /perf — 성능 병목 진단

**읽기 전용 명령: 코드 수정·커밋 절대 금지.**

## 대상: $ARGUMENTS

## 단계

1. **병렬 점검** — `perf-reviewer`와 `code-tracer` 에이전트를 동시에 투입:
   - `perf-reviewer`: N+1 쿼리, JPA fetch 전략, 풀스캔, 커넥션풀 고갈, 데드락, 인덱스 누락, 페이지네이션 없는 대용량 조회
   - `code-tracer`: 대상 기능의 전체 호출 경로 추적, 쿼리 실행 지점 식별

2. **개선안 우선순위**:
   - 🔴 즉시 개선 (사용자 체감 레이턴시·OOM 위험)
   - 🟠 배포 전 개선 권장
   - 🟡 백로그

3. **각 개선안마다**: 문제 지점(`파일:라인`), 원인, 구체적 개선 방법, 예상 효과.

> 진단 완료 후 /plan으로 개선 계획 수립 권장.
