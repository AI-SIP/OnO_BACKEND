# /commit — 커밋 단위 제안

**실제 커밋 실행 안 함. 제안만.**

## 맥락: $ARGUMENTS

## 단계

1. `git diff HEAD`와 `git status`로 현재 변경 전체 파악.

2. 변경을 논리 단위로 분할:
   - 각 단위는 독립적으로 빌드 가능하고 단일 목적.
   - 관련 없는 변경이 섞이지 않도록.

3. 각 단위별 커밋 메시지 제안:
   ```
   [태그] 한 줄 요약
   - 상세 1
   - 상세 2
   ```
   태그: `[Feat]` / `[Fix]` / `[Refactor]` / `[Chore]` / `[Test]` / `[Docs]` / `[Perf]`

4. 제안된 순서대로 커밋하는 명령어도 함께 제시.
