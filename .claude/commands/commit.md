# /commit — 커밋 생성

**변경 범위를 확인한 뒤 논리 단위별로 `git add`와 `git commit`까지 수행한다.**

## 맥락: $ARGUMENTS

## 단계

1. `git diff HEAD`와 `git status`로 현재 변경 전체 파악.

2. 변경을 논리 단위로 분할:
   - 각 단위는 독립적으로 빌드 가능하고 단일 목적.
   - 관련 없는 변경이 섞이지 않도록.

3. 각 단위별 커밋 메시지 결정:
   ```
   [태그] 한 줄 요약
   - 상세 1
   - 상세 2
   ```
   태그: `[Feat]` / `[Fix]` / `[Refactor]` / `[Chore]` / `[Test]` / `[Docs]` / `[Perf]`

4. 관련 없는 변경이 섞이지 않도록 단위별로 staging 대상을 분리한다.

5. 각 단위마다 `git add` 후 `git commit`을 실행한다.

6. 완료 후 생성된 커밋 해시, 커밋 메시지, 포함 파일을 보고한다.
