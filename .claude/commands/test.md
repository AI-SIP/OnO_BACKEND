# /test — 테스트 작성

## 대상: $ARGUMENTS (미지정 시 현재 git diff 변경분)

## 단계

1. **기존 테스트 품질 확인** — 대상 도메인의 기존 테스트를 먼저 읽는다.  
   좋으면 관례를 따르고, 그렇지 않으면 새로 깔끔하게 작성.

2. **`test-writer` 에이전트 투입** — 아래 원칙으로 테스트 작성:
   - test 프로필 + 격리 DB(H2 또는 전용 test DB) 사용.
   - **사용자 격리 필수**: 다른 사용자의 데이터에 접근할 수 없음을 반드시 검증.
   - 권한·필터·검색 조건의 모든 경우(정상/경계/실패) 망라.
   - 기존 테스트 복붙 금지 — 독립적으로 작성.
   - 단위 테스트 + DB 연동 테스트 위주. 외부 연동(FCM 등)은 mock.

3. **해당 테스트만 실행** — `./gradlew test --tests "대상클래스"` 실행 후 결과 확인.  
   실패 시 원인 파악 및 수정.

4. **통과 확인** — 모든 작성 테스트가 통과하면 결과 보고.
