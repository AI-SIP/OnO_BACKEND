# /report — 결과 보고서

## 기준: $ARGUMENTS (커밋 해시)

## 단계

1. **변경 분석** — `code-tracer` 에이전트로 `<커밋>..HEAD` 범위의 변경 내용 분석:
   - 수정된 파일·레이어 목록
   - 각 변경의 목적과 영향
   - Before/After 비교 가능한 핵심 지점

2. **`report-writer` 에이전트로 보고서 작성**:
   - 존댓말, 핵심 위주, Before/After 표 포함
   - 테스트 코드 내용 제외 (결과만)
   - 외부에 공유 가능한 수준으로

3. **저장 위치 확인** — 보고서 파일명(`<개발사항>_보고서.md`)을 제안하고 저장 위치를 사용자에게 물어본 뒤 저장.
