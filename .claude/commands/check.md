# /check — 다관점 검증

**읽기 전용 명령: 코드 수정·커밋 절대 금지.**

## 대상: $ARGUMENTS (미지정 시 현재 git diff)

## 단계

1. **전문가 병렬 검증** — 아래 에이전트를 동시에 투입:
   - `permission-auditor`: 사용자별 오답노트 데이터 격리, 인증·인가 누락 검증
   - `perf-reviewer`: N+1, JPA 풀스캔, 커넥션풀, 인덱스 누락 검증
   - `logic-reviewer`: 엣지/널, 동시성, 예외·롤백, 비즈 로직 정합성 검증
   - `db-safety`: FCM 실발송 경로, 잘못된 MySQL 쓰기, 데이터 오염 검증
   - **마이그레이션 파일이 포함되면**: `migration-safety` 추가 투입
   - **DTO·API 엔드포인트 변경이 포함되면**: `api-contract-checker` 추가 투입

2. **2차 반증** — 각 에이전트 결과를 교차 검토:
   - A가 발견한 문제를 B가 놓쳤는가?
   - 에이전트 간 충돌하는 판단이 있는가? → 재검토.

3. **위험도 분류 보고**:
   - 🔴 **즉시 수정 필요** (사용자 영향, 데이터 오염, 보안)
   - 🟠 **배포 전 수정 권장** (성능 저하, 예외 미처리)
   - 🟡 **다음 이터레이션** (마이너 개선)
   - 🟢 **이상 없음**

근거는 반드시 `파일:라인`. 추정 금지.
