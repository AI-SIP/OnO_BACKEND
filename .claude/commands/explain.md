# /explain — 코드 흐름·원리 설명

**읽기 전용 명령: 코드 수정·커밋 절대 금지.**

## 대상: $ARGUMENTS

## 단계

1. **`code-tracer` 에이전트** — 대상의 진입점부터 전체 호출 경로 추적:
   - Controller → Service → Repository → 쿼리 레이어
   - 데이터 흐름, 변환 지점, 분기 조건

2. **흐름 설명**:
   - 요청이 어떻게 처리되는가 (순서대로)
   - **왜 이렇게 설계했는가** (도메인 이유, 성능 이유, 제약 조건)
   - 주요 클래스·메서드의 역할 (`파일:라인` 포함)

3. **다이어그램** (선택) — 흐름이 복잡하면 텍스트 시퀀스 다이어그램 추가.
